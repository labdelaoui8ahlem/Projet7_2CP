Prerequisites for Using the Tool

What is an architectural tactic?

An architectural tactic is a software design choice that helps improve a system’s quality attributes like performance, availability, or security. Tactics are small, reusable strategies that can be combined to form larger architectural solutions.

What does it mean XPG ?

An XPG (Extended Parsing Grammar) is a way to describe the structure of a sequence diagram in a formal grammar format. It includes rules and relationships (like who calls who) and is used to generate JavaCC parsers that can analyze execution traces.

What is an execution trace ?

An execution trace is a file that logs the method calls (or function executions) that happened during a program’s run. It’s used to understand the behavior of a system and to detect patterns like architectural tactics.

What is the role of UML sequence diagrams ?

A UML sequence diagram visually shows how objects interact over time. It represents method calls between components, highlighting the order of interactions. These diagrams are the basis for generating grammar rules in your system.

What is parser in this context ?

A parser is a program generated by JavaCC that reads an execution trace and checks whether it follows the structure defined by a specific XPG grammar. In your tool, each parser corresponds to one architectural tactic (e.g. Ping/Echo). It scans the uploaded .txt file and looks for specific patterns of method calls that match the grammar — confirming whether the tactic is present or not.

What is JavaCC?

JavaCC is a tool that generates lexical analyzers and parsers from grammar specifications. It is used here to build tactic-specific parsers from XPG grammar rules.
